# 食材推荐输出格式

> 用户说"家里有 X Y Z 能做啥"时，agent 输出的格式。

---

## 用你手头的 {X, Y, Z} 我推荐这 3 道

### 1. <菜名 1>  <小标签>

- **命中**：X, Y（2/3）
- **还缺**：A、B（都是常备调料，问题不大）
- **难度**：简单 · 耗时：~25 分钟
- 本地有详细步骤 ✅ → 要走菜直接说一声

### 2. <菜名 2>  <小标签>

- **命中**：X, Z（2/3）
- **还缺**：C（可以跳过）
- **难度**：中等 · 耗时：~40 分钟
- 本地有详细步骤 ✅

### 3. <菜名 3>  <小标签>

- **命中**：Y, Z（2/3）
- **还缺**：D、E、F（D 可以用 F 替，E 得买）
- 本地**没有**完整步骤 — 我可以现场抓一份给你

---

## 规则

1. **至少命中 2 个**手头食材才推荐（SKILL.md 规则 #4）。
2. **还缺少的明着说**，不要藏。
3. 如果推荐里有本地已有的（`search.py` 能直接找到文件），标 ✅ 节省时间。
4. 如果本地都没 ≥2 命中的 → 推荐"需抓"的 2 道 + 建议 `fetch.py`。
5. 本地**一个都没命中** → 诚实说"数据库还不够丰富，让我先抓一批"，跑 `api_themealdb.py --bootstrap`。

## 给 agent 的决策顺序

```
user: "家里有 X Y Z 能做啥"
  ↓
python scripts/suggest.py --have "X,Y,Z" --min-match 2
  ↓
有 ≥3 命中 ≥2 的候选？
  → 输出 3 道（按命中数降序）
有 1-2 个候选？
  → 输出候选 + 建议 "或者我抓几道试试"
无候选？
  → 说明本地数据不够
  → 建议 python scripts/api_themealdb.py --bootstrap
  → 或按其中某个食材定向抓：fetch.py --source themealdb --ingredient <X 的英文>
```
