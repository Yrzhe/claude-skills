# Jackson Shuttleworth

**Group Product Manager, Retention Team** | **Duolingo** | Expertise: retention, growth, product-management, experimentation, habit formation

## Bio
Jackson Shuttleworth leads retention product work at Duolingo, with a focus on one of the company’s most powerful growth loops: streaks. His perspective matters because he has helped turn streaks from a simple gamification mechanic into a deeply tested system for habit formation, user commitment, and long-term DAU growth.

What makes Jackson distinctive is his unusually experimental, metrics-driven approach to retention. He combines behavioral psychology with relentless A/B testing, and he is willing to challenge conventional product instincts when the data says a simpler, clearer, or more intentional design will work better.

## Signature Frameworks

### Unit of Use Streak Design
- **When to use**: When designing a streak or habit mechanic and deciding what action should count.
- **Steps**:
  1. Identify the core unit of use in the product.
  2. Tie the streak to that meaningful action.
  3. Test whether simplifying the requirement improves retention.
  4. Avoid making the unit so small that it rewards low-intent behavior.
- **Example**: Duolingo moved from an XP-based streak to “do one lesson a day,” which better matched real usage and improved DAUs.

### 0-to-7 Day Loss Aversion Ramp
- **When to use**: When onboarding users into a streak or habit loop.
- **Steps**:
  1. Find the streak length where loss aversion starts to matter.
  2. Focus experiments on getting users there quickly.
  3. Reduce early failure risk with support and flexibility.
  4. After that threshold, protect consistency and meaning.
- **Example**: Duolingo found retention rises sharply from day 1 to day 7, so it optimized heavily for helping users reach a 7-day streak.

### Minimal V1, Then Iterative Expansion
- **When to use**: When validating a new feature idea.
- **Steps**:
  1. Strip the idea to the smallest testable version.
  2. Launch quickly.
  3. Measure the core hypothesis.
  4. Add complexity only after a win.
- **Example**: Duolingo first tested a simple 30-day streak message before later exploring goal lengths, opt-outs, and UI variants.

### Intentional Commitment UX
- **When to use**: When you want users to feel ownership over a goal.
- **Steps**:
  1. Use explicit choice language.
  2. Add real yes/no or choose-your-goal decisions.
  3. Test whether choosing increases commitment.
  4. Preserve useful friction.
- **Example**: Changing “Continue” to “Commit to My Goal” was a major win.

### Flexibility Without Cheapening
- **When to use**: When adding forgiveness to a habit system.
- **Steps**:
  1. Add limited flexibility like freezes.
  2. Test how much helps versus hurts.
  3. Give more flexibility early.
  4. Constrain it later so the streak still feels earned.
  5. Reward perfection visibly.
- **Example**: Two streak freezes beat one, but three did not beat two; later, Duolingo added “Perfect Streak” to reward users who avoided freezes.

### Clarity-First Streak Communication
- **When to use**: When users may not understand the mechanic.
- **Steps**:
  1. Assume misunderstanding.
  2. Explain the rules simply.
  3. Prioritize explicitness over cleverness.
  4. Make every visual element reinforce the cadence.
- **Example**: A short explanation of how streaks reset, plus a more calendar-like UI, improved comprehension and retention.

### Celebrate the Desired Behavior
- **When to use**: When you want to deepen emotional attachment to a habit.
- **Steps**:
  1. Identify emotionally important moments.
  2. Use animation, haptics, and visual treatment.
  3. Slow users down enough to feel the reward.
  4. Celebrate both continuity and perfection.
- **Example**: Haptics and celebratory animations made streak moments more rewarding and improved outcomes.

### Behavior-Based Notification Timing
- **When to use**: When sending reminders tied to recurring behavior.
- **Steps**:
  1. Use actual behavior as the main signal.
  2. Send reminders before the likely next session.
  3. Add last-chance reminders only when something meaningful is at risk.
  4. Frame notifications as protection, not just re-engagement.
- **Example**: Duolingo found that reminders sent 23.5 hours after the prior session outperformed user-set reminder times.

## Core Advice
- **When retention is weak and you’re considering streaks**: first verify the product already delivers enough value to bring users back — because streaks amplify existing demand; they don’t fix a broken core experience.
- **When deciding what counts toward a streak**: use the product’s true unit of use — because meaningful streaks build real habits, while tiny actions can reward low-intent behavior.
- **When early retention is the problem**: optimize aggressively for days 0–7 — because that’s where loss aversion starts to form and retention gains are steepest.
- **When testing a new retention mechanic**: launch the smallest version that isolates the hypothesis — because it speeds learning and avoids overbuilding.
- **When onboarding feels passive**: make the user explicitly commit — because ownership and intentionality improve downstream engagement.
- **When users drop streaks too easily**: add limited forgiveness early — because fragile habits need help before they become valuable.
- **When adding more flexibility seems tempting**: stop at the point where it starts to cheapen the streak — because forgiveness has diminishing returns.
- **When users seem confused**: over-invest in clarity — because even simple mechanics are often misunderstood.
- **When a feature is technically used but emotionally flat**: celebrate the moment — because emotional reinforcement strengthens the habit loop.
- **When timing reminders**: use revealed behavior, not stated preferences — because actual usage is a better predictor of the next session.

## Contrarian Takes
- **Conventional**: Engagement is secondary to core product value → **Their view**: retention is foundational — because users can’t benefit if they don’t return.
- **Conventional**: Less friction is always better → **Their view**: some friction creates commitment — because intentional choices increase ownership.
- **Conventional**: If an experiment wins, ship it → **Their view**: sometimes don’t ship winning experiments — because they may clutter sacred surfaces or weaken the product’s purpose.
- **Conventional**: More streak forgiveness always helps → **Their view**: flexibility has a ceiling — because too much trains users to skip.
- **Conventional**: Simple mechanics need little explanation → **Their view**: clarity is essential — because users often misunderstand even basic streak rules.
- **Conventional**: User preferences are the best personalization signal → **Their view**: behavior is better — because revealed actions outperform stated settings.
- **Conventional**: Late-night reminders are spammy → **Their view**: they can be welcomed — because users appreciate help protecting something they value.
